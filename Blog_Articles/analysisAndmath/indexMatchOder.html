The formula

=INDEX(range1,MATCH(lookup,range2,-1))


Explanation

To get entries from a table where lookup values are entered in descending order, use the INDEX and MATCH function, The MATCH is structured to estimate pairing using the -1 match type. In the illustration shown the formula in G6 is

=INDEX(D4:D8,MATCH(G5,C4:C8,-1))

In this case, entries in the array C4: C8 are arranged in descending order


Example

Mr. Timothy sells commodity X in cartoons for $250. A cartoon contains 50 units of Commodity X. 40 units will be $200. If a client requires 30 units of commodity X, this means $150. In this situation, we will desire to evaluate and return the next entry level basically in multiples of $50


How this formula works

For entries arranged in descending order, the formula estimates a match using the -1 match type. The MATCH component of the formula is scheduled as shown below:


=MATCH(G6,C4:C8,-1)

With the lookup logic as scheduled in cell G6, the MATCH function searches for the first entry in C4: C8 which is bigger or equates the lookup value. If a match is verified, MATCH finds its pair in the corresponding row. In the situation that a match is absent, MATCH searches through the range in C4: C8 till a smaller value is seen, it pulls back to return the initial row number. In the illustration, the lookup entry is 30, the row number of 3 is returned by MATCH to INDEX


=INDEX(D4:D8,3)

In the array D4: D8, the INDEX function replaces the third entry which is $150
